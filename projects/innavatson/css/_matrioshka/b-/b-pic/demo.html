Один спрайт на весь проект?

С точки зрения клиентской оптимизации, нужно сокращать количество запросов к серверу за внешними объектами, например, такими как изображения.
В лучшем случае на сайте должен быть всего один запрос к серверу за графикой. Чтобы решить эту задачу может ипользоватся техника CSS Sprites.
.b-pic предлогает универсальный подход для добавления изображений c использованием этой техники.

Как это работает?

Нарезаем всю графику в спрайт, подключаем стили b-pic.css, b-pic.ie.css, прописываем путь до основного спрайта в обоих файлах, ставим .b-pic на тот элемент у которого будет добавлена картинка, потом добавляем модификатор.

Пример:

HTML 

<a class="b-pic b-pic_fruit" href="#">Груша</a>

CSS 

.b-pic_fruit:before,
.b-pic_fruit .ie-before {
    content: '';

    width: 35px;
    height: 36px;
    margin-right: 5px;
}

.b-pic_fruit:before {
    background-position: 0 -128px;
}
.b-pic_fruit .ie-before ins {
    left: 0;top: -128px;
}

Добавляем размеры картинки и прописываем позицию картинки в спрайте для современных браузеров и для IE.

Преимущества

  • Кроссбраузерность
  • Универсальность
  • Не дублируем CSS код под каждую картинку
  • Нет экстраразметки в DOM
  • Минимальное количество запросов к серверу
